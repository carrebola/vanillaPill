ESLint con plantillas literales

22 de agosto de 2023 · 2 min de lectura

Para que ESLint reconozca y comprenda la sintaxis de HTML dentro de tus archivos JavaScript, especialmente en plantillas literales, puedes utilizar el plugin eslint-plugin-html. Este plugin agrega soporte para resaltar y analizar el código HTML dentro de tus plantillas literales. Aquí tienes los pasos para configurarlo:

Instalar el plugin:

Abre una terminal en la carpeta raíz de tu proyecto y ejecuta el siguiente comando para instalar el plugin eslint-plugin-html:

css Copy code npm install eslint-plugin-html --save-dev Configurar ESLint:

En tu archivo de configuración de ESLint (por lo general, .eslintrc.js o similar), agrega 'plugin:html/recommended' a la propiedad extends y agrega 'html' a la propiedad plugins. También puedes personalizar las reglas según tus necesidades. Aquí hay un ejemplo de configuración:

javascript Copy code module.exports = { // Otras configuraciones...

extends: ['plugin:html/recommended'],

plugins: ['html'],

// Otras reglas...

rules: { // Otras reglas... } }; Utilizar plantillas literales con html como etiqueta:

Una vez configurado el plugin, puedes utilizar la etiqueta html en tus plantillas literales sin que ESLint muestre errores. Aquí tienes un ejemplo:

javascript Copy code const template = html`

Home

Welcome to my website!

`;

Ejecutar ESLint:

Ahora puedes ejecutar ESLint normalmente en tu proyecto para que analice y verifique la sintaxis de HTML en tus plantillas literales. Ejecuta el siguiente comando en la terminal:

Copy code npx eslint . Asegúrate de que estás en la carpeta raíz de tu proyecto cuando ejecutes este comando.

Con la configuración y el plugin adecuados, ESLint debería ser capaz de reconocer y analizar el código HTML dentro de tus plantillas literales sin mostrar errores. Recuerda ajustar las reglas de ESLint según tus preferencias y necesidades.

Autenticación con Google

20 de agosto de 2023 · 4 min de lectura

ChatGPT Hasta mi último conocimiento en septiembre de 2021, Supabase ofrece autenticación mediante proveedores de terceros, como Google, utilizando el protocolo OAuth2. Para usar el proveedor de Google en Supabase con la API de JavaScript, debes seguir estos pasos generales:

Configurar el proveedor en Google Developer Console:

Ve a la Consola de Desarrolladores de Google. Crea un proyecto o selecciona uno existente. Configura las credenciales OAuth 2.0 para tu aplicación web. Deberás proporcionar las URLs de redireccionamiento autorizadas para tu aplicación, que generalmente serán proporcionadas por Supabase cuando configures la autenticación. Configurar la autenticación en Supabase:

Inicia sesión en el tablero de Supabase. Navega a tu proyecto y selecciona la pestaña "Authentication" en el menú lateral. En la sección "Providers", habilita el proveedor de Google e ingresa las credenciales y la información que obtuviste del paso anterior. Integrar la autenticación en tu aplicación:

Instala la biblioteca de Supabase JavaScript en tu proyecto:

bash Copy code npm install @supabase/supabase-js Importa y configura Supabase en tu código:

javascript Copy code import { createClient } from '@supabase/supabase-js';

const supabase = createClient('YOUR_SUPABASE_URL', 'YOUR_SUPABASE_ANON_KEY'); Utiliza la función signIn de Supabase para iniciar sesión utilizando el proveedor de Google:

javascript Copy code // Iniciar sesión con el proveedor de Google const { user, error } = await supabase.auth.signIn({ provider: 'google', });

if (error) { console.error('Error al iniciar sesión:', error.message); } else { console.log('Inicio de sesión exitoso:', user); } Manejar el resultado de la autenticación:

Después de que el usuario inicie sesión con éxito, user contendrá la información del usuario autenticado. Recuerda que la implementación específica puede variar según las actualizaciones y cambios en la plataforma Supabase. Te recomiendo consultar la documentación oficial de autenticación de Supabase y la documentación de la API de JavaScript de Supabase para obtener información actualizada y ejemplos detallados.

ChatGPT Configurar las credenciales OAuth 2.0 para tu aplicación web en Google Developer Console implica crear un proyecto en la consola y configurar las credenciales para permitir que tu aplicación se autentique a través de Google. Aquí hay un resumen de los pasos que debes seguir:

Accede a Google Developer Console:

Ve a la Consola de Desarrolladores de Google y asegúrate de haber iniciado sesión con tu cuenta de Google.

Crea un proyecto:

Si es la primera vez que usas la consola, es posible que necesites crear un nuevo proyecto. Si ya tienes proyectos, puedes seleccionar uno existente. Habilita la API de Google:

En la página de inicio del proyecto, haz clic en "Explorar y habilitar APIs". Busca y selecciona la API que deseas habilitar. Por ejemplo, si estás configurando la autenticación de Google, necesitas habilitar la API de "Google+ API" (que proporciona información de perfil del usuario). Crea credenciales:

En la página de la API, haz clic en la pestaña "Credenciales" en el menú lateral. Haz clic en el botón "Crear credenciales". Selecciona "ID de cliente de OAuth" como el tipo de credencial. Configura la información de la aplicación:

Selecciona "Aplicación web" como el tipo de aplicación. Proporciona un nombre para la credencial. Agrega las URLs de redireccionamiento autorizadas para tu aplicación. Estas son las URLs a las que Google redirigirá después de que un usuario inicie sesión correctamente. Por lo general, estas URLs serán proporcionadas por la plataforma en la que estás implementando la autenticación (en este caso, Supabase). Obtén las credenciales:

Después de configurar la credencial, Google te proporcionará un ID de cliente y un secreto de cliente. Estos son valores importantes que necesitarás para configurar la autenticación en tu plataforma, como Supabase. Usa las credenciales en tu aplicación:

Utiliza el ID de cliente y el secreto de cliente que obtuviste para configurar la autenticación en tu plataforma. En el caso de Supabase, proporciona estos valores en la configuración del proveedor de Google en el panel de autenticación de Supabase.

Es importante seguir los pasos con atención y configurar correctamente las URLs de redireccionamiento, ya que la autenticación OAuth2 se basa en redireccionar a los usuarios a esas URLs para procesar el flujo de autenticación.

Recuerda que los detalles exactos y la interfaz de Google Developer Console pueden cambiar con el tiempo, por lo que te recomiendo consultar la documentación oficial de Google y las instrucciones específicas de tu plataforma para obtener la información más actualizada.

Buenas prácticas creando mensajes 'commit'

14 de agosto de 2023 · 3 min de lectura
Carlos Arrebola Montávez
Profesor frontend fpllefia

Es importante escribir mensajes de commit claros y descriptivos al trabajar con Git y GitHub. Los mensajes de commit ayudan a ti y a otros colaboradores a entender los cambios realizados en el proyecto a lo largo del tiempo.

Aquí tienes algunos consejos para escribir mensajes de commit efectivos:

  1. Mantén los mensajes cortos y concisos: Los mensajes de commit deben ser breves pero informativos. Intenta resumir los cambios en una oración o dos.

  2. Usa el imperativo: Escribe los mensajes en tiempo presente y en imperativo. Por ejemplo, en lugar de "Agregado soporte para función X", escribe "Agregar soporte para función X". Esto hace que el mensaje sea más claro y directo.

  3. Explica el qué y el porqué: Describe qué cambios introduces en el commit y, si es relevante, por qué realizaste esos cambios. Esto ayuda a los demás a comprender el contexto de tus modificaciones.

  4. Separa el resumen del cuerpo: Si un commit requiere una explicación más detallada, coloca un resumen breve (máximo 50-72 caracteres) en la primera línea y luego agrega un espacio en blanco seguido de un párrafo o más de explicación adicional.

  5. Utiliza las palabras clave: Puedes usar palabras clave al comienzo de tus mensajes para indicar el tipo de cambio que realizaste. Algunas palabras clave comunes son "Agrega", "Corrige", "Modifica" y "Elimina".

  6. Sé específico: Evita mensajes genéricos como "Arreglos varios" o "Cambios menores". Es mejor ser específico sobre los cambios exactos que realizaste.

  7. Referencia los problemas: Si tu proyecto utiliza sistemas de seguimiento de problemas (como GitHub Issues), puedes hacer referencia a los números de problema en tus mensajes de commit. Por ejemplo, "Arregla #123" vinculará el commit con el problema número 123.

  8. Revísalo antes de confirmar: Antes de confirmar tus cambios, tómate un momento para revisar el mensaje de commit y asegurarte de que sea claro y preciso.

  9. Mantén un estilo consistente: Si trabajas en un equipo, es útil mantener un estilo de mensajes de commit consistente en todo el proyecto. Esto facilita la lectura y comprensión de los cambios a lo largo del tiempo.

  10. No tengas miedo de editar: Si te das cuenta de que un mensaje de commit no es claro o no sigue las pautas, no dudes en editar el mensaje antes de hacer push.
